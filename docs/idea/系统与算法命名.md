# Cogito — 系统命名与算法设计原理

## 一、系统名称：Cogito

**词源**：拉丁语 *cogito*，意为"我思"。出自笛卡尔 *Cogito, ergo sum*（我思故我在）。

**选名理由**：

| 维度 | 说明 |
|------|------|
| 语义契合 | 本系统的核心创新是让 AI agent 拥有**持久的内部认知状态**并通过反思驱动演化——"我思"正是对这一过程的最精炼表达 |
| 学术辨识度 | 单词短、好记，在 AI/ML 领域无重名冲突，适合做项目名和论文标题 |
| 词根关联 | cogito → cognition（认知）→ cognitive state（认知状态），与核心概念 `CognitiveState` 同源 |
| 哲学呼应 | 笛卡尔用"我思"确立主体存在性；本系统用"认知状态"确立 agent 的内在连续性——从"无状态的工具调用"到"有内在心智的自主体" |

**项目路径**：`forks/cogito`

## 二、核心算法名称：MC-ESES

**全称**：Monte Carlo Endogenous State Evolution Search

**中文**：蒙特卡洛内生状态演化搜索

### 命名拆解

| 组成部分 | 含义 | 对应的设计决策 |
|----------|------|---------------|
| **Monte Carlo** | 蒙特卡洛方法 | 使用 UCB 选择 + 随机模拟 + 回传的树搜索框架，继承 MCTS 的探索-利用平衡机制 |
| **Endogenous** | 内生的 | 状态演化由 agent 内部反思驱动（自由能最小化），而非外部搜索策略强加——区别于 aira-cogito 中 prompt 拼装的"外生"状态 |
| **State Evolution** | 状态演化 | 搜索对象是认知状态 `z_t` 的演化轨迹，不是方案 Node 本身；Node 是状态的外显投影 `a_t = E(z_t)` |
| **Search** | 搜索 | 在认知状态空间上进行系统性探索，寻找最优的认知演化路径 |

### 与标准 MCTS 的关键区别

```
Standard MCTS                          MC-ESES
─────────────────────────────────────────────────────────────
节点 = 独立方案 (code + metric)         节点 = 认知状态 z_t
搜索 = 在方案空间中找最优解             搜索 = 在认知演化轨迹中找最优路径
展开 = 生成一个新方案                   展开 = 从 z_t 外显化多个候选动作 a_t^(k)
评估 = 单一 metric                     评估 = 多维反馈 r_t（含新颖性、稳定性、自由能降）
回传 = 更新方案的 Q 值                  回传 = 更新认知演化轨迹的质量估计
节点间无因果关联                        节点间有内生的状态演化因果链
```

## 三、核心设计原理

### 3.1 根本洞察：从方案搜索到认知演化

aira-cogito 的搜索范式是：

```
Node_t → Operator → Node_t+1
```

每个 Node 是独立的方案（plan + code + metric），搜索策略在方案空间中选择和变异。LLM 的"理解"分散在 prompt 拼接中，是隐式的、无结构的、用完即弃的。

Cogito 的范式是：

```
z_t → E(z_t) → a_t → execute → r_t → U(z_t, r_t) → z_t+1
```

搜索对象从方案提升为认知状态 `z_t`，方案 `a_t` 只是状态的外显投影。

### 3.2 认知状态 `z_t` 的结构

```python
class CognitiveState:
    task_understanding: str           # 对任务的结构化理解
    working_hypotheses: List[str]     # 当前活跃的工作假设
    search_preferences: Dict          # 搜索方向偏好（如"特征工程" vs "模型替换"）
    experience_summary: str           # 从历史尝试中压缩的经验
    failure_patterns: List[str]       # 已识别的失败模式
    confidence_map: Dict[str, float]  # 对各方向的信心估计
```

`z_t` 独立于任何 Node 存在。Node 是 `z_t` 的副产品，`z_t` 才是被持续维护和演化的核心对象。

### 3.3 内生演化动力学

状态更新遵循自由能最小化原则：

```
z_t+1 = z_t - η ∇F(z_t; r_t, m_t)
```

其中：
- `F` — 变分自由能，衡量"认知状态与观测反馈的不一致程度"
- `r_t` — 本轮执行反馈（不只是 metric，还含错误模式、进退趋势）
- `m_t` — 记忆/先验（结构化的历史经验积累）
- `η` — 学习率（控制状态更新幅度）

实际实现中，`U(z_t, r_t)` 通过一次 LLM 反思调用完成：输入当前 `z_t` 和结构化反馈 `r_t`，输出更新后的 `z_t+1`。

### 3.4 MC-ESES 四步流程

```
1. SELECTION    — UCB 选择最有演化潜力的认知状态节点 z_t
                  Q_i = 状态演化潜力（非方案价值）
                  UCB: z_t = argmax_i { Q_i + c √(ln N / n_i) }

2. EXPANSION   — 从 z_t 外显化 K 个候选动作：a_t^(k) ~ E(z_t)
                  每个动作是一个 plan + code 方案
                  同一个 z_t 可产出不同风格的方案

3. EVALUATION  — 执行 a_t^(k)，获得多维反馈 r_t^(k) = O(a_t, D)
                  继续内生演化：z_t+1^(k) = U(z_t, r_t^(k))
                  可沿轨迹继续演化多步

4. BACKPROP    — 将轨迹质量回传至祖先节点
                  G_i = Σ γ^t φ(r_t)
                  R_t 包含：新颖性、可解释性、稳定性、自由能降
```

### 3.5 与 aira-cogito 的映射关系

| aira-cogito 概念 | Cogito 对应 | 关系 |
|----------------|------------|------|
| Node (plan + code + metric) | 外显动作 `a_t = E(z_t)` | Node 从搜索对象变为状态的投影 |
| Operators (Draft/Improve/Debug) | 外显器 `E` 的具体实现 | 保留复用，但由 `z_t` 驱动选择 |
| MetricValue (score + is_buggy) | 多维反馈 `r_t` 的子集 | 扩展为含错误模式、趋势分析 |
| Search Policy (UCT/fitness) | 状态更新 `U(z_t, r_t)` | 从"选方案"变为"演化认知" |
| Journal (节点列表) | 记忆 `m_t`（先验/经验） | 从扁平记录变为结构化知识积累 |

## 四、命名体系总览

| 层级 | 名称 | 说明 |
|------|------|------|
| **系统/项目** | Cogito | 整体框架，含认知状态层 + 搜索算法 + 执行基础设施 |
| **核心算法** | MC-ESES | 在认知状态空间上的蒙特卡洛树搜索 |
| **核心数据结构** | CognitiveState (`z_t`) | 持久化的内部认知状态对象 |
| **核心函数** | StateUpdater `U(z_t, r_t)` | LLM 驱动的反思式状态更新 |
| **外显映射** | Executor `E(z_t)` | 从认知状态生成可执行方案 |
| **论文方法名** | Cogito + MC-ESES | *Cogito: Endogenous Cognitive State Evolution for AI Research Agents* |

## 五、实施阶段

### 阶段 A — 最小验证（在 aira-cogito 上加层）

- 引入 `CognitiveState` 对象
- 增加反思步骤 `U(z_t, r_t)`
- 用状态 enrich operator prompt
- 在 AIRS-bench 上对比"有/无显式状态"

### 阶段 B — 完整重构（独立 Cogito 系统）

- 搜索节点从 Node 替换为 `CognitiveState`
- 实现完整 MC-ESES 四步流程
- 多维反馈 `r_t` 和自由能评估
- 在认知状态空间上进行 MCTS/Evolutionary 搜索
